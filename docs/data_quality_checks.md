# Проверки качества данных

## Проверки тикетов

- уникальность `ticket_id`;
- заполненность ключевых полей;
- корректный порядок дат;
- отсутствие отрицательного времени ответа и решения;
- корректность флага `sla_breached`.

## Проверки событий

- уникальность `event_id`;
- наличие связанного `ticket_id`;
- корректность `event_time`;
- порядок событий внутри обращения.

## Проверки измерений

- уникальность ключей справочников;
- наличие отдела у каждого агента;
- наличие сегмента у каждого тикета;
- наличие даты обращения в календаре.

## Где смотреть результат

Автоматическая сводка сохраняется в `data/processed/data_quality_summary.csv`. sql-версия проверок лежит в `sql/data_quality_checks.sql`.
